## Load and test in SBCL

```lisp
(ql-dist:install-dist "http://dwightaspencer.com/distinfo.txt" :prompt nil)
(ql:quickload :DwightASpencerCom)
;; Self instance and corpus KB are auto-initialised at load time.
```

## Refresh corpus to latest live build

```lisp
;; Pull the latest CI-packaged snapshot:
(ql:update-dist "dwightaspencercom" :prompt nil)
(ql:quickload :DwightASpencerCom)

;; Or fetch corpus.lisp directly from the site (dexador auto-loaded via ASDF):
(DwightASpencerCom:load-live-corpus)
(DwightASpencerCom:all-posts)
```

## Package structure

The namespace follows Java/Clojure reverse-domain convention.
`DwightASpencerCom` encodes the site domain `dwightaspencer.com`.

```
com.dwightaspencer/logic    — micro-Prolog (db-assert, db-prove-all, unification)
com.dwightaspencer/corpus   — post facts (assert-post-facts, make-site-kb,
                               find-post, find-by-tag, load-live-corpus)
com.dwightaspencer/render   — PostScript backend (render :ps/:postscript stream kb)
DwightASpencerCom           — top-level ql:quickload target (finger, query, kb,
                               render, all-posts, load-live-corpus)
```

Local nicknames within each layer:
- `com.dwightaspencer/corpus` uses `logic` as a short alias for `com.dwightaspencer/logic`
- `com.dwightaspencer/render` uses `corpus` as a short alias for `com.dwightaspencer/corpus`

## Corpus

`corpus.lisp` is Hugo-generated — never edit directly.
Lives at site root `/corpus.lisp`, not under `/lisp/`.
Regenerates on every `hugo --gc`.

The `+corpus-url+` constant in the corpus template points to:
`https://dwightaspencer.com/corpus.lisp`

### Bundled stub vs CI snapshot

The tarball contains `corpus-stub.lisp` compiled as `corpus`.
The CI build pipeline (GH Actions) replaces it with the Hugo-generated
`corpus.lisp` before packaging via `redist compile` — the deployed tarball
always contains the live post facts snapshotted at build time.

When running from the stub directly (not the tarball), the KB is empty.
Use `load-live-corpus` to populate it from the live site.

## Updating the release

Source files are in `hugo/static/lisp/`. The CI pipeline handles packaging:

1. Hugo builds → `public/corpus.lisp` generated
2. CI stages `hugo/static/lisp/*.lisp` + `public/corpus.lisp` into a local git repo
3. `redist compile` generates `releases.txt`, `systems.txt`, `distinfo.txt`, and the tarball
4. Output is copied to `hugo/public/lisp/`

No manual checksum computation needed — redist handles it.

## Known issues

| Symptom | Cause | Fix |
|---|---|---|
| DESTRUCTURING-BIND error with `<!doctype` | URL wrong, returns HTML | Use `http://dwightaspencer.com/distinfo.txt` |
| `System "DwightASpencerCom" not found` | Dist not installed or cache stale | `(ql:update-all-dists)` then retry |
| `all-posts` returns NIL | Stub corpus loaded, not CI tarball | `(ql:quickload :dexador)` then `(DwightASpencerCom:load-live-corpus)` |
| `corpus.lisp` 404 | Hugo not generating it or wrong URL | Check `hugo/layouts/index.lisp` output format |
